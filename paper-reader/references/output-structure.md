# 论文解读输出结构

## 六大模块（固定顺序）

### 0. 论文基本信息
- 标题、作者、发表年份/会议/期刊
- 一句话概括（≤40字）

### 1. 数学与背景知识准备
列出理解本文**必须掌握**的前置知识，每条包含：
- 概念名称
- 一句话解释
- 本文中的具体使用场景

### 2. 研究问题 & 动机
- **要解决什么问题**：现有方法的具体痛点
- **为什么重要**：应用价值或理论意义
- **核心假设**：作者的关键前提

### 3. 方法论详解
- **整体思路**：2-3句话描述核心 idea
- **关键步骤**：分点说明主要组成
- **关键公式**：最核心的 1-3 个公式 + 白话解释
- **与已有方法的区别**

### 4. 实验结果 & 数据
- **数据集**：用了哪些，规模如何
- **评价指标**：用什么衡量，为什么
- **主要结论**：关键数字/对比（表格或列表）
- **消融实验**：验证了哪些设计决策（如有）

### 5. 优点、局限性 & 潜在改进方向
| 维度 | 内容 |
|------|------|
| 主要贡献 | 作者声称的 + 你的判断 |
| 局限性 | 论文承认的 + 你观察到的 |
| 潜在改进 | 基于局限性可以怎么做 |

---

## Word 文档结构

Word 文档与 Markdown 内容一致，额外在**封面页后**插入一张论文框架图，图示论文的核心逻辑链：

```
[ 研究问题 ] → [ 核心方法 ] → [ 主要贡献 ]
```
