# 分析报告写作模板

Claude 撰写报告时，严格按照以下结构和写作规范输出。

---

## 报告结构（固定顺序）

```
# 数据分析报告：{文件名}

## 0. 数据概览
## 1. 数据质量检查
## 2. 分布分析
## 3. 趋势分析（若有时间列）
## 4. 相关性分析（若有 ≥3 个数值列）
## 5. 关键发现与建议
```

---

## 各模块写作规范

### 0. 数据概览

必须包含以下信息（数字来自 summary.json）：

```markdown
| 项目 | 值 |
|------|----|
| 总行数 | X 行 |
| 总列数 | X 列 |
| 数值列 | 列举列名 |
| 类别列 | 列举列名 |
| 时间列 | 列举列名（若无则填"无"） |
```

一句话描述数据集的业务含义（若无法判断则写"数据集业务背景不明，以下分析基于数据结构"）。

---

### 1. 数据质量检查

列出所有列的缺失情况：

```markdown
| 列名 | 缺失数量 | 缺失率 | 建议 |
|------|---------|--------|------|
| col_a | 0 | 0% | ✅ 无需处理 |
| col_b | 120 | 24% | ⚠️ 建议核查 |
| col_c | 310 | 62% | ❌ 严重缺失，慎用 |
```

**缺失率判断标准**：
- < 5%：✅ 无需处理
- 5% ~ 30%：⚠️ 建议核查
- > 30%：❌ 严重缺失，慎用

---

### 2. 分布分析

每个图表按如下格式嵌入：

```markdown
### 2.1 {列名} 分布

![柱状图](output/bar_{列名}.png)

**解读**：{2-3句，说明分布形态、最高/最低类别、是否存在明显集中趋势}
```

写作要点：
- 用具体数字支撑观点，如"占比最高的类别为 X，达到 42%"
- 指出异常值或意外发现
- 避免泛泛而谈，如"分布比较均匀"后必须说明均匀的程度

---

### 3. 趋势分析（有时间列时必须包含）

```markdown
### 3.1 {指标} 随时间变化趋势

![折线图](output/line_{列名}.png)

**解读**：{说明整体趋势方向（上升/下降/波动）、峰值和谷值出现的时间点、是否有周期性规律}
```

---

### 4. 相关性分析（数值列 ≥ 3 时必须包含）

```markdown
### 4.1 数值变量相关性矩阵

![热力图](output/heatmap.png)

**强相关对（|r| > 0.7）**：
- {变量A} 与 {变量B}：r = {值}，{一句话业务解读}

**中等相关对（0.4 < |r| ≤ 0.7）**：
- （若有则列出，若无则写"无显著中等相关"）

**注意**：相关性不等于因果性，以上发现需结合业务背景进一步验证。
```

---

### 5. 关键发现与建议

固定格式：

```markdown
## 5. 关键发现与建议

### 关键发现
1. {最重要的发现，一句话}
2. {第二重要的发现}
3. {第三重要的发现（若有）}

### 建议
- **数据层面**：{数据质量问题的改进建议，若无问题则不写此项}
- **分析层面**：{建议下一步做什么分析，如"建议对 X 列做时间分解分析"}
- **业务层面**：{若能判断业务背景，给出 1-2 条可操作建议}
```

---

## 写作禁忌

- ❌ 不写"数据显示情况良好"这类无信息量的话
- ❌ 不在没有图表支撑的情况下做趋势判断
- ❌ 不把 summary.json 里的原始 JSON 直接贴进报告
- ❌ 不跳过缺失值严重的列（必须标注）
- ✅ 每个数字结论后面必须有来源（来自哪张图或哪个统计量）
